# Deep-dive audit — July 2026

An honest snapshot of where the product stands against its own ambition:
a top-quality iRacing overlay that competes at the highest level of the
current market. Produced from a full read of the code and docs, a green
build (694/694 tests, warnings-as-errors), fresh `render.ps1` output, and a
live web check of every competitive claim we make. The actions that came
out of it live in [ROADMAP.md](ROADMAP.md); this file is the evidence and
the judgement, dated so it can be re-run against a future state.

## Verdict

The engineering discipline this repo claims for itself is real, and it is
the product's genuine moat: measured numbers over modelled ones, sim data
over invented schemes, features deleted when they fail legibility, every
calculator tested. The strategy layer (pit-exit projection, push-or-save,
traffic forecast) is still differentiated — no overlay draws these. Three
things stand between "excellent, principled overlay" and "competes at the
top": the market picture we planned against was missing its fastest-moving
front (free, open-source overlays — irDashies above all); accessibility is
currently zero options in a hobby whose demographics make colour-vision
deficiency common; and a handful of product-surface gaps that users feel
daily — no resize, five panels on by default, two of which overlap on first
run, and an unsigned installer. All are fixable, none require abandoning
the "race engineer, not dashboard" identity — they require finishing it.

## What's verifiably strong

- **The tree is green and the claims check out.** Build clean under
  `TreatWarningsAsErrors`, 694/694 tests passing, and the tuning defaults,
  guards and behaviours documented in FEATURES.md match the code (one stale
  coordinate aside, below).
- **Nothing on screen is invented.** Every readout traces to a sim-published
  variable, a measurement, or arithmetic over those — see the accuracy table
  below. The two features that failed this bar historically (CPI safety chip,
  catch/defend column) were removed or parked, which is the discipline
  working, not failing.
- **The renders hold up against the reference product.** Typography at
  correct optical sizes, tabular numerals, one-meaning-per-hue, chips as a
  single shared object — the standings and relative read as professional
  timing surfaces, not hobby panels.
- **The unglamorous foundations exist**: installer + silent auto-update that
  never restarts mid-session, single-instance guard, settings that survive
  updates, dev/live isolation, off-screen layout rescue, sim-gated
  visibility. Competitors get this wrong more often than the marketing
  suggests.

## Corrections to our own record

Found by checking our claims against the world, in the spirit of "the code
wins":

1. **The market snapshot missed the free/open-source front.**
   [irDashies](https://irdashies.com) — free, open-source, iRacing-only —
   ships 20+ widgets including standings, relative, fuel, two track-map
   forms, weather/wind, a blind-spot monitor, a **rejoin indicator**, a
   **slow-car-ahead warning**, input traces, sector deltas, driver tagging,
   and drag-to-resize with snapping. It is the fastest-moving competitor and
   it costs nothing, which dissolves "free" as a differentiator on its own.
   The SimHub ecosystem (e.g. the benofficial2 overlay pack) is a second
   free front. ROADMAP's snapshot now includes this.
2. **"Pit-exit projection — nobody else has it" was too strong.**
   [CrewChief](https://thecrewchief.org) — free, and installed at every
   level of the sport — has spoken pit-exit position predictions since
   v4.9.3.0 ("where will we be after a stop?"), including opponent pit-exit
   warnings. The honest claim, now in ROADMAP: **no overlay draws it**, and
   ours differs in kind — CrewChief needs a benchmark stop practised in
   advance and answers on request by voice; ours learns the lane cost from
   the whole field live, updates continuously, and states the cost it used.
   That is still a real advantage. It is not "nobody else has this".
3. **The traffic-forecast landscape sharpened.** iOverlay's multiclass
   indicator and irDashies' "faster cars from behind" both warn that quicker
   traffic is coming. Ours remains the only one that says **where and when**
   (lap + sector). The claim survives, correctly narrowed.
4. **FEATURES.md documented a stale fuel-widget default position**
   (`80,140`; the code says `600,24`). Fixed in this pass.
5. **The first-run layout overlaps.** Standings at `Left=24` is 616px of
   panel plus margins — its panel spans to x≈654. The fuel widget's panel
   starts at x≈614. On a fresh install the fuel widget sits ~40px over the
   standings' right edge until the user drags it. Small, but it is the
   first thing a new user sees. Fix queued in ROADMAP (code change, so not
   in this docs-only pass).

## Accuracy: is anything on screen speculative?

The user-facing test: every number must be one the sim published, one we
measured, or arithmetic over those — and where it degrades, it must degrade
visibly rather than guess.

| Readout | The number is | Verdict |
|---|---|---|
| Standings gaps / interval / laps-down | Differences of the sim's `CarIdxF2Time`; laps-down derived from time, not lap counts | Sim-published, transformed |
| Position-change arrows | Class position latched at first sight of each car | Measured; the mid-race-launch caveat is documented and inherent (no SDK grid variable) |
| Strength of Field | iRacing's own formula | Published formula |
| **Projected iRating** | Community-reconstructed Elo with a calibrated 200-point constant | **The one modelled number on screen.** Market-standard (RaceLab and iOverlay show the same projection), zero-sum enforced, docs state it's good to a few points. No evidence the sim's formula changed. Keep, keep the caveat. |
| Relative deltas | Sim's `CarIdxEstTime`, corrected across the line | Sim-published; the **120s fallback lap time** (used only when the roster lacks class pace) is the weakest constant in the app — rarely hit, worth remembering |
| Traffic forecast | Measured gap + sim's per-class pace + sim's sector boundaries | Sim-published inputs; the class-pace assumption is stated in docs and checkable a lap later |
| Fuel burn / margin / add / save-to | Rolling measured burn + arithmetic | Measured. **But "Add" assumes one more stop covers the race** — in a 2+-stop race it can name more litres than the tank holds. Least-honest number on the panel in endurance; queued in ROADMAP |
| Race laps remaining | Sim lap count, or time ÷ measured lap rounded up | Sim-published / measured |
| Pit loss ("costs 29s") | Median of the whole field's stops, both pit-road edges required, bounds reject non-stops | Measured, and shown on-widget so the projection is falsifiable |
| Pit-exit projection | Player's `F2Time` + learned loss, counted against the field | Measured + sim-published; projects the field as it stands, which the strip's wording owns |
| Save cost | Least-squares over the driver's own laps; sign checked; refuses to read past the observed burn range | Measured, and refuses more often than it reports — correct |
| Delta bar | The sim's own `LapDeltaToBestLap` | Sim-published; we add hold-at-the-line, not a second opinion |
| Radar placement | Track shape learned from the player's driving; defers to the sim's spotter and **dims** when lateral position is unknowable | Measured; admits ignorance instead of drawing it |
| Track map | Learned outline, closure-corrected; the player's line, said so in docs; learning % drawn until ready | Measured; honest about what it is |
| Setup strip | Sim's `DriverSetupName` / `IsModified` | Sim-published |

**Conclusion: no speculative readouts.** Two watch items (iRating model,
120s fallback), both documented, neither worth removing. The multi-stop
"Add" gap is an *arithmetic honesty* item, not a speculation item, and is
now on the roadmap.

## Usefulness: does anything fail the decision test?

Widget by widget: standings, relative, fuel and radar are core and each
changes in-car decisions. The track map earns its place (clear air, where a
rival went) but is the least decision-dense of the five — that's a defaults
question, below, not an existence question. The delta is correctly opt-in.
The setup strip prevents a race-losing mistake and switches off cleanly.

**No novelty widgets found.** The weakest passenger is the **manufacturer
badge** — an identification aid, not a decision aid. It is already off by
default and tagged experimental, with genuine open research questions in
ROADMAP; if the relative-widget research answers "no", the honest end state
is deletion rather than a permanent toggle, per the repo's own rule. The
gimmick tier (heart rate, G-force, boost) remains correctly out.

## Accessibility

Today the app has **no accessibility options**. The audit below says the
baseline is much better than that sentence sounds — but at "compete at the
highest level", with a user base that skews heavily male, red-green colour
vision deficiency alone is ~8% of users. No mainstream overlay ships a
colour-vision mode; this is an open differentiator that fits our
"right by default" identity exactly.

### Colour vision — signal-by-signal

The hue-discipline ("every colour means one thing") accidentally produced
real redundancy: almost every colour-coded signal carries a second channel,
so a colour-blind driver today loses **salience** (which row lights up),
rarely **information**:

| Signal | Second channel | Under CVD |
|---|---|---|
| Position arrows `▲2`/`▼1` | glyph carries the direction | Safe |
| Projected-iRating chip | arrow | Safe |
| Fuel margin badge | "LAPS SPARE" / "LAPS SHORT" label | Safe |
| Tank gauge | tick position vs fill | Safe |
| Delta readout + bar | sign, and the bar grows left/right from centre | Safe |
| Standings Last column | signed value | Safe |
| Incident count amber→red | the `2x/17x` text itself | Safe |
| Flag chip | the flag's name in the chip | Safe |
| License chip | the letter + SR in the chip | Safe |
| PIT / DAMP / MOD | words | Safe |
| Class banners (standings) | class name-plate text | Safe |
| Traffic strip | the whole sentence | Safe |
| **Fastest-lap purple** | none | **Hue-only** (it is also iRacing's own convention, which a CVD racer already can't use in-sim) |
| **Lapped/lapping name tint** (red/blue) | none | **Hue-only**; blue survives, the red side fades toward white/grey |
| **Class colour bars/dots** (relative, track map) | none | **Hue-only**, but the hues are the sim's own (`CarClassColor`) — inventing replacements would break the familiarity that made us adopt them |
| Radar: green you / red danger glow | position (centre vs sides) | Function survives; **salience drops** — protan vision dims red specifically, which is the worst place to lose brightness |

The core pair is the measured problem: `Positive #33D689` and
`Negative #FF5C6C` sit at nearly identical relative luminance (~0.62 vs
~0.56), so under deuteranopia/protanopia they collapse to similar muddy
tones distinguishable by neither hue nor brightness.

**Recommendation (now in ROADMAP): one "colour-blind friendly palette"
setting.** Not per-deficiency modes — one preset that re-solves the hue
budget (gain/loss to a blue/orange axis, fastest-lap and lap-tint values
re-picked, radar glow to a protan-bright hue), leaving the sim-supplied
class and license colours alone (the license chip already carries its
letter). Every meaning-hue lives in `App.xaml` as a single-source resource,
so this is a contained piece of design work plus a resource swap — the
architecture already paid for it.

### Other disabilities

- **Photosensitivity: safe.** The only flashing element (setup reminder)
  pulses at ~0.7 Hz as a soft amber wash — far below the 3–30 Hz hazard
  band — and has an off switch. The radar glow fades continuously, never
  blinks.
- **Low vision: decent.** UI scale to 175% globally and per-widget; text
  contrast on the panel material is comfortably high (near-white on
  near-black); weights deliberately run heavy to counter greyscale AA.
  Drag-to-resize (roadmap) is also the path past the 175% cap.
- **Deaf / hard-of-hearing: a genuine strength nobody states.** Spotters —
  iRacing's own and CrewChief — are audio. A visual proximity radar built on
  the sim's own spotter signal *is* the accessible spotter. Worth saying in
  the README when the radar is described, because it is true and it matters
  to real users.
- **Motor: acceptable, one gap.** Everything is mouse-only; widgets position
  by drag with no keyboard nudge, and precision placement at 40px overlap
  tolerance is fiddly. Low priority, noted.
- **Screen readers: out of scope for the overlays** (they are visual
  instruments over a visual sim), and the settings window is standard-ish
  WPF that mostly works; verify focus visuals survived the control
  retemplating whenever that window is next touched.

## Defaults out of the box

Current: five of six widgets on (all but delta). `HideWhenSimClosed`
defaulting to on already delivers desktop-level unintrusiveness — nothing
appears until the sim runs. The question is over-the-sim intrusiveness on
first contact: today a new user gets **four visible panels at once**
(standings, relative, fuel, track map — the radar correctly hides itself
until a car is near, so it costs nothing).

**Recommendation (in ROADMAP): the track map joins the delta in
`DefaultOffWidgets`.** It is the least decision-dense panel, it does not
self-hide, and its constituency (multiclass, endurance, unfamiliar tracks)
knows to want it — one tray click away. Standings + relative + fuel is the
right first impression: the race-engineer core, nothing twice. The radar
stays on because a self-hiding widget has no out-of-the-box cost. The
first-run fuel/standings overlap gets fixed in the same pass.

## Customisation

The stance ("the default *is* the product", column matrices are the
competitors' moat and our anti-feature) survives this audit — with more
force, in fact, now that irDashies gives full customization away free:
**competing on customization is a fight we cannot win and should not
enter. Competing on judgement is an open lane.** What changes:

- **Drag-to-resize gets promoted.** It was "carried-over polish"; it is
  actually the largest daily-feel gap against every competitor (both paid
  tools and irDashies resize; we offer four fixed scale steps). Resize is
  not layout-editor customization — it is making one opinionated layout fit
  any monitor.
- **A small set of *subtractive* content toggles is compatible with the
  ethos** — remove what a given driver doesn't act on, never rearrange what
  remains. The first real candidate: **hide the ratings chips (license +
  iRating)** — a known constituency races better without rating anxiety on
  screen, which is a judgement-shaped reason, not a preferences-shaped one.
  The existing per-widget toggles (badges, pace trend, setup strip) already
  follow this pattern.
- **Per-session-type profiles** (already mid-term) remain the
  customization axis that fits the product: the practice layout genuinely
  isn't the race layout.
- **Still out, on purpose:** column pickers, reordering, per-column widths,
  fonts/themes, a layout editor.

## The market, honestly (July 2026)

Three fronts plus two adjacencies:

- **RaceLab** (free + €4.90/mo Pro): the breadth platform — 29 overlays,
  7 sims, VR, layout library, streaming tools. Wins on surface area.
- **iOverlay** (free + €2.50–4.95/mo Pro): iRacing-only, sells
  customization — pick-your-columns standings, pit entry/exit helper,
  behind-only traffic warning, VR, Twitch integration.
- **irDashies** (free, open source): 20+ widgets, resize/snapping, driver
  tagging, weather, blind-spot monitor, rejoin indicator, slow-car-ahead —
  shipping fast, and free removes price as our differentiator.
- **Adjacent — CrewChief** (free voice engineer): spoken pit-exit
  prediction (benchmark-based), spotter, strategy answers on request. Not
  an overlay; absolutely a substitute for parts of our strategy layer.
- **Adjacent — Garage 61 / iRacePlan**: endurance team stint/fuel planning
  with live timing. Serves the *team* use-case we've declared a non-goal.

Where we genuinely lead: the strategy layer as *drawn, live, self-checking
readouts* (learned pit loss stated on the widget, save-vs-stop priced in
seconds, meeting-point forecast by sector); a no-database track map; and
the right-by-default discipline itself. Where "top of market" is decided
beyond widgets, in rough order of user-felt weight: **resize; sane
first-run defaults; a signed installer** (SmartScreen's "unknown publisher"
is a real trust tax on every download; certificates cost real money and are
noted in ROADMAP as a distribution item, not a code item); and a public
identity (site, community presence) — real, but out of scope for this
audit.

## Ranked recommendations

Docs items landed in this pass; code items queued in ROADMAP, none started:

1. **Correct the record** — market snapshot + uniqueness claims + stale
   coordinate. *(Done in this pass.)*
2. **Defaults pass** — track map into `DefaultOffWidgets`; fix the
   first-run fuel/standings overlap. One-line-ish changes, outsized
   first-impression value.
3. **Colour-blind friendly palette** — one preset, hue budget re-solved in
   `App.xaml`, sim colours untouched. No mainstream overlay has this.
4. **Drag-to-resize** — promoted from polish to the top of the feel gap.
5. **Multi-stop honesty on the fuel widget** — never name more litres than
   the tank holds; say "+1 stop" when that's the truth.
6. **Weather forecast strip** — unchanged priority; irDashies shipping
   weather widgets confirms demand, ours should stay decision-shaped
   (what's it doing in 20 minutes, not current-conditions wallpaper).
7. **Research candidates from prior art** — rejoin indicator and
   slow-car-ahead both pass the decision test hard (they prevent the two
   worst incident types); study irDashies' versions before designing.
8. **Sign the installer** when distribution ambitions firm up.
